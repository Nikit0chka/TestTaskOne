Данный проект представляет собой реализацию тестового задания на платформе**Avalonia**с использованием подхода**Clean Architecture + DDD**и паттерна**MVVM**. Приложение отображает данные о взвешивании машин (урожайность/грузоперевозки) в таблице и графике, позволяет добавлять новые записи, редактировать существующие, ассинхронно работает с базой данных, читает файл в фоновом режиме. Основные события логируются в консоль и файл с использвоанием Serilog

## 🏛 Архитектура
Проект построен по принципам**Clean Architecture**с разделением на слои:
- **Domain**— сущности, value objects, интерфейсы репозиториев (бизнес-логика, DDD).    
- **Application**— use cases, DTO, маппинг, валидация, сервисы приложения.    
- **Infrastructure**— реализация репозиториев, контекст EF, миграции, логгирование, чтение файлов.    
- **Presentation**— Avalonia UI, ViewModels, Views, ресурсы. 
**Основные принципы:**
- Внедрение зависимостей (DI) через`Microsoft.Extensions.DependencyInjection`.    
- Асинхронность во всех операциях с БД и файловой системой.    
- Разделение ответственности (SOLID).

**Ключевые файлы:**

- `WeightingDbContext`— контекст EF.    
- `MainWindowViewModel`— главная ViewModel
- `AddRecordViewModel`— ViewModel для добавления записи.
- `Repository`— работа с машинами.    
- `FileImportService`— фоновая загрузка из файла.
